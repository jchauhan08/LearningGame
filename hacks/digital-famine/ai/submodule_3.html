Welcome to AI APIs

An AI API (Application Programming Interface) is a set of rules that allows developers to integrate artificial intelligence capabilities into their applications without building AI models from scratch.

APIs act as bridges, enabling software to communicate with and use AI services by sending requests and receiving responses.

Understanding how a complete application works helps you build better AI integrations. Let's explore the microblog system as an example.

Layer 1: UI Framework (opencs.html)

Provides the structure, layout, and dependencies for the microblog panel.

  • ‚Ä¢ Includes CSS and JavaScript libraries (Tailwind, jQuery, DataTables)
  • ‚Ä¢ Creates the sliding panel structure with HTML elements

  • ‚Ä¢ Adds floating toggle button and close button
  • ‚Ä¢ Implements basic open/close functionality for the panel
  • ‚Ä¢ Checks page frontmatter (microblog: true) to activate features

Layer 2: UI Functionality (microblog_foundation.html)

Makes the microblog interactive by handling user actions and data display.

  • ‚Ä¢ Manages modal window for creating/editing posts
  • ‚Ä¢ Handles form submission and data collection
  • ‚Ä¢ Renders the table of microblog posts dynamically
  • ‚Ä¢ Implements filter modes (page-specific vs all posts)
  • ‚Ä¢ Binds event handlers to buttons (create, edit, filter)
  • ‚Ä¢ Calls API functions when data needs to be fetched or saved

Layer 3: API Management (microblog.js)

Handles all communication between the frontend and backend server.

  • ‚Ä¢ Provides three core functions: fetchPosts(), createPost(), updatePost()
  • ‚Ä¢ Sends HTTP requests to server (GET, POST, PUT methods)
  • ‚Ä¢ Formats data as JSON before sending to server
  • ‚Ä¢ Receives and parses JSON responses from server
  • ‚Ä¢ Handles errors during network communication
  • ‚Ä¢ Abstracts server communication so UI layer doesn't need to know details

üíª JavaScript UI Functionality

UI functionality refers to the JavaScript code that makes your web page interactive. Let's explore the three main components:

1. Modal System (Pop-up Window)

A modal is a pop-up window that appears over the main page for creating and editing posts.

function openModal({ mode, post = {} }) {
  document.getElementById('microblog-modal').classList.remove('hidden');
  document.getElementById('modal-title').textContent = 
    mode === 'edit' ? 'Edit Microblog Post' : 'Create Microblog Post';
  document.getElementById('post-id').value = post.id || '';
  document.getElementById('content').value = post.content || '';
}

What this does:

  • ‚Ä¢ Makes the modal visible by removing the 'hidden' CSS class
  • ‚Ä¢ Changes the title based on create/edit mode
  • ‚Ä¢ Pre-fills form with existing data (edit) or leaves empty (create)

2. Form Submission (Saving Data)

When a user fills out the form and clicks "Save":

document.getElementById('microblog-form').onsubmit = async function(e) {
  e.preventDefault();
  
  const id = document.getElementById('post-id').value;
  const content = document.getElementById('content').value;
  
  try {
    if (id) {
      await updatePost({ id, content });
    } else {
      await createPost({ content });
    }
    closeModal();
    renderMicroblogTable();
  } catch (err) {
    alert('Error saving post: ' + err.message);
  }
};

Step-by-step:

  • 1. e.preventDefault() stops page refresh
  • 2. Collect data from form fields
  • 3. Check for ID - if exists, update; if not, create
  • 4. Call appropriate API function
  • 5. Close modal and refresh table display

3. Table Rendering (Displaying Data)

This function fetches post data and displays it in a table:

async function renderMicroblogTable() {
  const container = document.getElementById('microblog-playground');
  
  try {
    // Fetch data from API
    const data = await fetchPosts();
    
    // Build table HTML
    let tableHTML = '<table id="microblog-table">...';
    data.microblogs.forEach(post => {
      tableHTML += `<tr>
        <td>${post.userName}</td>
        <td>${post.content}</td>
      </tr>`;
    });
    tableHTML += '</table>';
    
    // Display table
    container.innerHTML = tableHTML;
    
    // Add interactive features
    $('#microblog-table').DataTable();
  } catch (error) {
    container.innerHTML = `<div style="color:red;">Error: ${error.message}</div>`;
  }
}

Why this matters: The table rebuilds from scratch every time data changes, ensuring the UI always matches current data.

üîå API Management Layer

The API Management layer (microblog.js) handles all communication between your application and the server. Let's explore the three core functions:

1. Fetching Posts (READ)

Retrieves posts from the server, optionally filtered by page.

export async function fetchPosts(pagePath) {
            let apiUrl = pythonURI + '/api/microblog';
            if (pagePath) {
                apiUrl += `?pagePath=${encodeURIComponent(pagePath)}`;
            }
            
            const response = await fetch(apiUrl, {
                method: 'GET'
            });
            
            return await response.json();
        }

What this does:

  • ‚Ä¢ Builds the API URL from base URI
  • ‚Ä¢ Adds page filter as query parameter if provided
  • ‚Ä¢ Uses GET method to retrieve data (not modifying)
  • ‚Ä¢ Waits for server response and parses JSON

2. Creating Posts (CREATE)

Sends a new post to the server to be saved.

export async function createPost(postData) {
            const apiUrl = pythonURI + '/api/microblog';
            
            const response = await fetch(apiUrl, {
                method: 'POST',
                body: JSON.stringify(postData)
            });
            
            return await response.json();
        }

Key differences from fetchPosts:

  • ‚Ä¢ Uses POST method (creating new data)
  • ‚Ä¢ Includes data in request body
  • ‚Ä¢ Converts JavaScript object to JSON with JSON.stringify()
  • ‚Ä¢ Server responds with created post including new ID

3. Updating Posts (UPDATE)

Sends updated post data to modify an existing post.

export async function updatePost(postData) {
            const apiUrl = pythonURI + '/api/microblog';
            
            const response = await fetch(apiUrl, {
                method: 'PUT',
                body: JSON.stringify(postData)
            });
            
            return await response.json();
        }

Key differences from createPost:

  • ‚Ä¢ Uses PUT method (updating existing data)
  • ‚Ä¢ postData must include ID to identify which post to update
  • ‚Ä¢ Similar structure but different HTTP method tells server this is an update

HTTP Methods Explained

Method Purpose Has Body?
GET Read/retrieve data No
POST Create new data Yes
PUT Update existing data Yes

Key concept: Different HTTP methods tell the server what operation to perform on the data.
